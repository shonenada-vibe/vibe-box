# Bring-up Checklist

## 目标

先把 `ESP32-S3-Touch-ePaper-1.54` V2 的关键板载资源逐一验证，避免在应用开发阶段同时调试多个硬件问题。

板子型号前提：

- `ESP32-S3-PICO-1-N8R8`
- 对应 Waveshare 文档中的 `V2`

## 验证顺序

1. USB 烧录与串口日志
2. 墨水屏基础显示
3. Wi-Fi STA 联网
4. SHTC3 温湿度读取
5. RTC 读取
6. 触摸输入
7. ES8311 音频录放

不要先做完整业务逻辑。先确认每个外设最小可用。

## 每项记录内容

- 使用的官方示例名称
- 使用的资源包版本
- 是否明确是 `V2`
- 编译所需 ESP-IDF 版本
- 关键引脚 / 总线
- 成功现象
- 失败现象
- 临时修复方式

## 需要最终确认的板级信息

### 显示

- e-paper 控制器型号
- 刷新方式
- 是否支持局刷
- 帧缓冲占用

### 音频

- ES8311 初始化流程
- 麦克风输入路径
- 扬声器输出路径
- I2S 采样率 / 位宽 / 声道配置

### 传感器

- SHTC3 所在 I2C 总线
- RTC 型号与地址
- 触摸控制器型号

### 电源

- 电池电量读取方式
- 低功耗模式影响
- 唤醒源

## 当前结论

- 项目主开发框架使用 `ESP-IDF`
- 服务端与固件分仓内协作，但职责分离
- 第一阶段先打通 `录音 -> 上传 -> 返回文本 -> 墨水屏显示`
